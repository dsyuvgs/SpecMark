## 〇、核心法则（背下来，一秒判类型）

### 黄金判定口诀

```
软件坏了 → 功能
软件能用但慢/卡 → 性能  
软件能用但不好用/服务差 → 体验
算法推荐不准/搜索不对 → 内容
```

### 一句话定义

| 类型 | 一句话定义 | 判断灵魂 |
|------|-----------|---------|
| **功能** | 软件本身坏了/出错了/不可用了 | "这个功能还能用吗？" → 不能 = 功能 |
| **性能** | 能用但速度慢/卡顿/耗电高 | "是卡/慢还是坏了？" → 卡/慢 = 性能 |
| **体验** | 能用但不好用/操作复杂/服务体验差 | "是软件的问题还是服务的问题？" → 服务 = 体验 |
| **内容** | 算法输出质量差（推荐/搜索/匹配） | "是算法给的结果不对吗？" → 对 = 内容 |

### 最关键的边界（必考！）

| 场景 | 归类 | 为什么 |
|------|------|--------|
| 配送超时/餐品凉了 | **体验** | 软件没坏，是配送服务体验差 |
| 骑手态度恶劣/扔餐 | **体验** | 骑手行为不是软件bug |
| 商家取消订单/食物有异物 | **体验** | 商家行为/食品质量≠软件故障 |
| 客服不解决/推诿/模板回复 | **体验** | 服务质量问题是体验维度 |
| 支付扣钱了但订单没生成 | **功能** | 支付功能真的坏了=数据不一致 |
| 闪退/崩溃/白屏 | **功能** | 软件完全不能正常工作 |
| 页面打不开/一直转圈 | **功能** | 加载功能失效（不是慢，是坏） |
| 按钮点了没反应 | **功能** | 交互功能失效 |
| 太卡了/加载很慢 | **性能** | 能用但速度不达标 |
| 发烫/耗电快 | **性能** | 物理资源消耗异常 |
| 操作太复杂/找不到入口 | **体验** | 功能能用但操作路径差 |
| 广告太多/推送关不掉 | **体验** | 商业化策略影响使用体验 |
| 会员太贵/配送费涨了 | **体验** | 价格感知属于体验 |
| 推荐越来越不准 | **内容** | 算法输出质量问题 |
| 搜索结果完全不匹配 | **内容** | 搜索算法输出质量问题 |
| 匹配机制不公平 | **内容** | 匹配算法输出质量问题 |
| "能不能加个XX功能" | **体验** | 建议类一律归体验 |

## 一、情绪判定

### 三分类

| 情绪 | 判定条件 | 示例 |
|------|---------|------|
| 负向 | 不满/失望/愤怒/功能失效/期望落空/趋势性恶化 | "太垃圾了""支付失败""越来越卡" |
| 中性 | 客观描述/委婉建议/不带情绪/保留态度 | "能不能加个深色模式""还行""凑合" |
| 正向 | 满意/认可/喜欢 | "超好用""推荐给大家""体验很好" |

### 关键规则

| 场景 | 判定 | 说明 |
|------|------|------|
| 反讽 | 负向 | "太棒了，一天闪退八次"→负向 |
| 正面词+但+问题 | 负向 | "好用是好用，就是偶尔闪退"→负向 |
| 委婉建议 | 中性 | "能不能加个XX"→中性 |
| 纯功能失效描述 | 负向 | "支付失败""闪退"→功能坏了本身就是负面 |
| 趋势性恶化 | 负向 | "越来越差""越来越卡"→负向 |
| 保留态度 | 中性 | "还行""凑合""一般般"→中性 |
| "不错但XX有问题" | 负向 | 有问题就判负向，不管前面多好听 |

## 二、问题类型判定（核心！）

### 2.1 功能问题 — 详细定义

**定义**：产品功能无法按预期正常运行，软件本身存在缺陷或错误。

**判定条件（满足任一即判功能）**：

| 子类 | 判定关键词 | 典型示例 |
|------|-----------|---------|
| 崩溃/闪退 | 闪退、崩溃、死机、白屏、黑屏、卡死、ANR、强制退出 | "每次打开就闪退""游戏崩了" |
| 功能失效 | 不能用、没法用、用不了、无响应、没反应、失效、失灵 | "按钮点了没反应""支付功能不能用" |
| 数据错误 | 丢失、没了、不见了、消失、错误、不对、乱码 | "订单消失了""聊天记录没了""金额算错了" |
| 连接失败 | 连不上、断连、掉线、离线、断开、连不上WiFi | "总是掉线""蓝牙连不上" |
| 登录/注册失败 | 登录不上、注册失败、验证码收不到、密码错误 | "验证码永远收不到""登录按钮点了没反应" |
| 支付异常 | 扣款失败、支付失败、重复扣款、扣了钱没订单 | "扣了299但会员没到账""付款后订单消失了" |
| 页面异常 | 打不开、空白、显示不出来、加载不出、404、报错 | "首页一片空白""详情页打不开" |
| 保存/同步失败 | 保存不了、同步失败、自动保存丢失 | "写了两个小时自动保存失败了" |
| 上传/下载失败 | 上传失败、下载失败、传不了、下不了 | "视频上传到99%就失败了" |
| 安全/隐私泄露 | 被盗、泄露、账号被盗、隐私暴露 | "账号被异地登录了""隐私被泄露了" |
| 权限异常 | 没权限、权限不够、被封、被限流、被禁言 | "明明有VIP结果说没权限" |

**排除清单（这些不是功能问题！）**：
- ~~配送超时~~ → 体验
- ~~骑手态度差~~ → 体验
- ~~商家取消订单~~ → 体验
- ~~客服不解决问题~~ → 体验
- ~~食物有异物/变质~~ → 体验
- ~~操作复杂/找不到~~ → 体验
- ~~广告太多~~ → 体验
- ~~价格贵~~ → 体验
- ~~建议加功能~~ → 体验

### 2.2 性能问题 — 详细定义

**定义**：系统能正常工作，但响应速度、流畅度、资源占用等物理指标低于用户期望或行业标准。

**判定条件（满足任一即判性能）**：

| 子类 | 判定关键词 | 典型示例 |
|------|-----------|---------|
| 卡顿 | 卡、很卡、太卡、特别卡、变卡、卡住、不流畅 | "刷视频的时候经常卡一下" |
| 速度慢 | 慢、太慢、很慢、启动慢、打开慢、响应慢、加载慢 | "APP启动要8秒""消息发送延迟3秒" |
| 延迟高 | 延迟、lag、460、高延迟 | "语音通话有明显延迟" |
| 耗电/发热 | 耗电、费电、掉电快、发烫、发热、过热 | "打一把游戏电量掉了30%""手机发烫严重" |
| 掉帧/帧率低 | 掉帧、帧率、不跟手、画面不流畅 | "开车场景明显掉帧" |
| 缓冲/转圈 | 缓冲中、一直转圈、loading、加载中 | "看视频前要缓冲10秒" |
| 内存/CPU占用高 | 占用高、内存高、CPU高、后台耗资源 | "挂着APP手机就很烫" |
| 安装包/更新包大 | 包太大、安装包大、更新包大 | "一个更新包要2GB" |

**关键区分**：
- "页面一直转圈**打不开**" → **功能**（最终结果是打不开=功能失效）
- "页面转圈**但最后打开了只是慢**" → **性能**（最终能用=慢）

### 2.3 体验问题 — 详细定义（最大的一类！）

**定义**：软件功能本身可用，但用户在使用过程中感到不便、不满或不愉快。
包括：操作体验、界面体验、服务体验、价格感知、商业化体验。

**判定条件（满足任一即判体验）**：

#### A. 操作/交互体验
| 关键词 | 示例 |
|--------|------|
| 麻烦/复杂/繁琐 | "安全验证太麻烦了""退款流程走了7天" |
| 找不到/难找 | "设置入口藏得太深了""找不到退出登录在哪" |
| 步骤太多/操作复杂 | "改个头像要5步""上传文件流程太长" |
| 不方便/不好用/难用 | "这个功能太难用了""一点都不方便" |
| 引导不足/看不懂 | "新用户根本不知道怎么用""没有新手引导" |

#### B. 服务体验（⚠️最容易误判为功能的区域！）
| 关键词 | 正确归类 | 示例 |
|--------|---------|------|
| 配送超时/慢/凉/洒 | **体验** | "外卖送到已经凉透了""配送超时30分钟" |
| 骑手/司机/快递员态度 | **体验** | "骑手态度特别差""司机绕路了" |
| 骑手/司机行为（扔门口/提前确认送达） | **体验** | "骑手把餐扔门口就跑了""提前点了送达但我还没收到" |
| 商家行为（取消/不承认/不管） | **体验** | "商家接单后又取消""商家不承认有问题" |
| 食物/商品质量（异物/变质/不新鲜） | **体验** | "吃出头发了""牛蛙肉变质的" |
| 客服问题（不管/不解决/推诿/模板/慢） | **体验** | "客服只会复制粘贴模板话术""投诉了没人处理" |
| 退款/售后慢/难 | **体验** | "申请退款等了一周还没到账""售后审核太慢了" |
| 评价/好评受限 | **体验** | "想给差评发现只能选好评选项" |
| 等待时间长（非软件加载） | **体验** | "等了20分钟没人接单""问诊等了一个小时才回三句话" |

#### C. 价格/费用感知
| 关键词 | 示例 |
|--------|------|
| 太贵/贵了/涨价 | "会员太贵了""配送费从3块涨到9块""皮肤价格越来越贵" |
| 性价比低 | "充了648全是保底""性价比太低了" |
| 不值这个价 | "年费188感觉不值" |

#### D. 商业化/广告体验
| 关键词 | 示例 |
|--------|------|
| 广告太多/关不掉 | "开屏广告5秒跳不过去""全是广告" |
| 推送太多/骚扰 | "每天早上收到三条营销推送" |
| 会员权益缩水 | "开了会员还是有好多歌听不了" |
| 自动续费陷阱 | "被自动续费了完全没提醒" |
| 诱导付费/套路 | "优惠券规则写得天花乱坠实际用不了" |

#### E. 建议/需求类（一律归体验！）
| 关键词 | 示例 |
|--------|------|
| 能不能加/希望增加/建议 | "能不能加个暗黑模式""希望能增加数据导出功能" |
| 想要/想要一个 | "我想要一个记账功能" |

#### F. 界面/设计体验
| 关键词 | 示例 |
|--------|------|
| 界面丑/布局乱/字体小 | "界面设计太土了""字体太小老人看不清" |
| 不美观/不好看 | "新版UI太丑了" |

### 2.4 内容问题 — 详细定义

**定义**：涉及算法输出质量、信息准确性、内容相关性的问题。

**判定条件（满足任一即判内容）**：

| 子类 | 判定关键词 | 典型示例 |
|------|-----------|---------|
| 推荐不准/不好/单一 | 推荐不准、推荐不好、推荐单一、推荐过时、推荐重复 | "推荐的全是不感兴趣的""天天推一样的" |
| 搜索不匹配/无关 | 搜不到、搜出来不对、搜索结果不相关、搜XX出YY | "搜Python入门出来全是Excel教程" |
| 算法/匹配问题 | 匹配不公平、匹配机制有问题、算法推荐 | "我一个钻石老是匹配到王者对手" |
| 翻译/字幕/歌词错误 | 翻译错、字幕不同步、歌词延迟、歌词翻译不对 | "字幕和声音对不上" |
| 信息质量差 | 内容低俗、内容过时、信息旧、攻略老/关门了 | "旅游攻略写的餐厅早关门了" |
| 图文不符/虚假 | 图文不符、描述不符、假货、刷评、水军 | "商品图片和实物完全不一样" |
| 分类/排序不合理 | 分类乱、排序不合理 | "歌曲分类太乱了" |

## 四、优先级判定

### 矩阵

| 情绪 | 类型 | 特殊条件 | 优先级 |
|------|------|----------|--------|
| 正向 | 任意 | - | 🔴低 |
| 中性 | 功能 | - | 🟡中 |
| 中性 | 性能/内容/体验 | - | 🟢低 |
| 负向 | 任意 | 严重故障(闪退/崩溃/数据丢失/支付失败/安全漏洞) | 🔴高 |
| 负向 | 任意 | 付费用户+任何问题 | 🔴高 |
| 负向 | 功能 | 核心功能失效 | 🔴高 |
| 负向 | 功能 | 非核心功能 | 🟡中 |
| 负向 | 性能 | 严重影响(发烫不敢用/完全卡死) | 🔴高 |
| 负向 | 性能 | 一般性能问题 | 🟡中 |
| 负向 | 内容 | 严重影响(假货/隐私泄露/诈骗) | 🔴高 |
| 负向 | 内容 | 一般内容问题 | 🟢低 |
| 负向 | 体验 | 严重影响(等超久/骑手恶劣/被自动续费/食品安全) | 🔴高 |
| 负向 | 体验 | 一般体验问题 | 🟢低 |

### 高优先级触发关键词

**功能类高优**：闪退、崩溃、死机、白屏、数据丢失、支付失败、安全漏洞、隐私泄露、账号被盗
**体验类高优**：等了X分钟/小时没人接单、骑手恶劣/扔餐/态度差、被自动续费没提醒、食品安全问题（异物/变质）、客服完全不理、发烫不敢用
**内容类高优**：假货/刷单/诈骗/隐私泄露/杀熟

### 缓解词降级规则

缓解词：偶尔、有时、不影响使用、但整体、但基本、不过、可以接受、还行、勉强

**例外（以下情况即使有缓解词也不降级）**：
- 闪退/崩溃/数据丢失/支付失败/安全漏洞 → 始终高
- 食品安全问题 → 始终高
- 被欺骗（自动续费/虚假宣传） → 始终高

### 单条反馈分类输出

```json
{
  "sentiment": "负向",
  "problem_type": "体验",
  "priority": "高",
  "confidence": 0.95,
  "reasoning": "配送超时属于服务体验问题，软件功能本身未失效"
```

### 字段说明

| 字段 | 取值 | 说明 |
|------|------|------|
| sentiment | 负向/中性/正向 | 用户情绪倾向 |
| problem_type | 功能问题/性能问题/体验问题/内容问题 | 问题归属类别 |
| priority | 高/中/低 | 处理优先级 |
| confidence | 0.0-1.0 | 分类置信度 |
| reasoning | string | 分类理由（一句话解释为什么这么分） |
